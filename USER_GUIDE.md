# FinanceTrack User Guide

This guide explains how to use every major feature in FinanceTrack.

---

## 1) First Login

1. Open the app URL.
2. Sign in with your account.
3. Go to **Settings** and change your password immediately.

---

## 2) Navigation

Main app sections:
- Dashboard
- Transactions
- Accounts
- Budget
- Forecast
- Life Events
- What-If
- Alerts
- Settings

---

## 3) Dashboard

Use Dashboard as the weekly command center.

What it shows:
- Net worth summary
- This month income/spending
- Accounts grouped by type
- Spending by category chart
- Forecast preview
- Upcoming life events
- Alert highlights
- Recent transactions

Recommended routine:
- Review dashboard weekly
- Drill into Alerts/Budget when warning cards appear

---

## 4) Transactions

Use this page to keep raw data accurate.

Actions:
- Add transaction
- Edit transaction
- Delete transaction
- Filter by date/account/category/search
- Import CSV/XLSX
- Export CSV

Import best practice:
1. Export from bank/card provider
2. Import monthly into FinanceTrack
3. Review duplicates/errors
4. Fix uncategorized rows

---

## 5) Accounts

Track assets and liabilities.

Actions:
- Create/edit/remove accounts
- Update balances
- Add balance snapshots

Best practice:
- Update balances monthly for key accounts
- Keep debt balances current for accurate projections

---

## 6) Budget

Budget compares planned spend vs actual.

Important:
- Set `budget_amount` for categories you want monitored.
- Over-budget categories can trigger Alerts.

Review cadence:
- Mid-month and end-of-month

---

## 7) Forecast

Forecast projects future cash/net-worth over selected months.

Uses:
- historical transactions
- recurring rules
- life event costs
- scenario adjustments

Use it to:
- detect future cash crunches
- compare plan options before committing

---

## 8) Life Events

Life Events model large planned financial changes.

Examples:
- move
- vacation
- major purchase
- loan payoff plan

Tips:
- Add realistic date ranges
- Use monthly breakdown when possible

---

## 9) What-If Scenarios

Scenarios let you safely model alternatives.

Workflow:
1. Clone baseline
2. Change transactions/rules/events in scenario
3. Compare scenario vs baseline
4. Keep only useful scenarios

---

## 10) Alerts

Alerts aggregates actionable issues.

Current alert types:
- Budget overrun alerts
- Upcoming event payment alerts

Use alerts to:
- catch overspending early
- prepare for near-term event costs

---

## 11) Settings

Use Settings to:
- change password
- maintain account security hygiene

---

## 12) Monthly Operating Checklist

1. Import last month transactions
2. Categorize uncategorized rows
3. Update balances/snapshots
4. Review budget overages
5. Review alerts
6. Check 3–6 month forecast
7. Update life events as needed

---

## 13) Common Issues

### API not loading
- Ensure backend service is running
- Ensure token/session still valid

### Import failure
- Validate CSV headers/date/amount formats
- Remove extra blank header rows

### No alerts
- Possibly healthy state
- Or no budgets/events configured for alert conditions
